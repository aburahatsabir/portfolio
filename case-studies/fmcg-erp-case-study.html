Auditability & Change Control

This architecture implements strict "ERP-lite" governance, differentiating it from standard automation scripts. All logical operations prioritize data integrity over flexibility.

Versioned Master Data

Price changes are recorded with effective dates, ensuring historical orders reference the correct snapshot, not the current live price.

Immutable Transactions

Every OrderID is sequentially generated and logged with timestamps and creator IDs. Delete operations are strictly disabled.

Corrections via Reversal

Errors in finalized orders must be corrected via "Credit Note" reversal entries, preserving the original audit trail—no overwriting allowed.

Daily Reconciliation

Automated nightly checklists verify that physical stock movement matches the ledger totals to catch discrepancies early.

Relational Logic Without a Database

Built using normalized tables and QUERY functions. This production formula generates on-demand invoices with sub-second lookups against the master dataset.
